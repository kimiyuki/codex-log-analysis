# AGENTS.md

## 返答と変更方針

- ユーザー向けの説明は日本語で簡潔に行う。
- 変更は最小差分を原則とし、不要な依存追加や大きな抽象化は避ける。
- 不明点が実装判断に影響する場合は、推測で埋めずに確認する。

## このリポジトリの目的

- `sessions_snapshot/` 配下の Codex セッション JSONL を terminal から分析する。
- まずは「日単位の振り返り」と「skill 化の種を拾う」ためのローカル CLI を整える。
- データの解釈は evidence-first で行い、ログにない値を勝手に補完しない。

## 実装ガイド

- ログ読取は fail fast を基本とする。
- 壊れた JSONL 行や読めないファイルは、外部データ境界として stderr に警告を出して継続してよい。
- セッション内部で期待する構造が欠けている場合は、黙ったデフォルト化ではなく「不明」として明示する。
- `title` や `branch` など、ログに安定して存在しない値は無理に生成しない。

## Python / uv

- Python 実装を使う場合は `uv` 前提で管理する。
- 依存追加は本当に必要になるまで行わず、標準ライブラリで足りるならそれを優先する。
- 実行例は `uv run python -m ...` または `uv run <script>` で README に残す。

## 検証

- 変更後は最低限、CLI のヘルプ表示とサンプル日付での実行を確認する。
- 可能なら `uv run python -m compileall src` のような軽量検証も行う。

## 情報源

- Codex の動作やデータ構造に疑問がある場合は、まず `sessions_snapshot/` と `state_*.sqlite` の実データを優先して確認する。
- それでも仕様確認が必要なときは、Codex 本体の source code（https://github.com/openai/codex）を参照してよい。
